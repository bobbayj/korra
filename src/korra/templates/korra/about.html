About What exactly is this for?

Sending files in a secure fashion sucks. There are a number of options out there, but all of them fail for one reason or another.

  • You can send a file by email, but emails are not encrypted, and pass through multiple untrusted hosts on their way to their destination.
  • You could send encrypted emails, but email encryption is fantastically user-hostile. 99.99% of people on the internet don't use it and don't understand it. It also requires installing special software on the origin and destination machines, something that is often not permitted in office environments.
  • You can put the file into an encrypted archive (.zip, etc.) and then email it to someone, but again, this requires special software on both hosts.
  • You could push the file onto a shared server somewhere that's designated "safe". This is alright, if you have such a server and/or your users can access it.

This project is a work around for this situation. I came up with it when I worked for the British government where IT solutions are fragmented across departments, workstations are locked-down, and crypto-literacy is variable.

The idea is this:

  1. You upload a file to this site using the web form.
    • The file is encrypted with a password of your choosing so that even the server administrator can't read it.
    • You can opt to auto-delete the file after a fixed number of hours, or have it deleted immediately after it's downloaded by your recipient.
  2. The site will give you a URL to give to your recipient, which they can click on to download the file. However downloading it will require the password you initially used. Send this password to the recipient by a means other than what you used to send the URL.
  3. The recipient clicks on the link, enters the password, and they download the file. No special software required, no tech savvy needed.
